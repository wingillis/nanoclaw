# Code Generator

Create complete code repositories with proper structure, git initialization, and best practices.

## Output Directory

All projects are created in `/workspace/extra/code/` (mounted from host).

Check if the code directory is available:
```bash
test -d /workspace/extra/code && echo "CODE_DIR_AVAILABLE" || echo "NO_CODE_DIR"
```

If `NO_CODE_DIR`, respond with:
> Code generation requires the main channel. Send your request there to create projects.

## Project Creation Workflow

### 1. Plan the project structure

Before creating files, outline:
- Project name (kebab-case)
- Language/runtime
- Key directories
- Entry points
- Dependencies

### 2. Create the project directory

```bash
PROJECT_NAME="my-project"
mkdir -p "/workspace/extra/code/${PROJECT_NAME}"
cd "/workspace/extra/code/${PROJECT_NAME}"
```

### 3. Initialize version control

```bash
git init
git config user.email "agent@nanoclaw.local"
git config user.name "NanoClaw Agent"
```

### 4. Create project structure by language

#### Node.js Projects

```bash
# Standard Node.js structure
mkdir -p src test docs
touch src/index.ts
touch test/index.test.ts
touch README.md
touch .gitignore

# Initialize package.json
npm init -y

# Common .gitignore for Node.js
cat > .gitignore << 'EOF'
node_modules/
dist/
.env
*.log
.DS_Store
EOF
```

#### Python Projects

```bash
# Standard Python structure with uv
mkdir -p src/myproject tests docs
touch src/myproject/__init__.py
touch src/myproject/main.py
touch tests/__init__.py
touch tests/test_main.py
touch README.md
touch .gitignore

# Initialize with uv (creates pyproject.toml automatically)
uv init --name myproject

# Create a virtual environment and install dependencies
uv venv
uv add pytest  # Example dev dependency

# Common .gitignore for Python
cat > .gitignore << 'EOF'
__pycache__/
*.py[cod]
*$py.class
.venv/
venv/
.env
*.egg-info/
dist/
build/
uv.lock
EOF
```

**Note:** `uv` is available for fast Python package management. Use `uv add <package>` to add dependencies and `uv run <command>` to run Python scripts in the virtual environment.

#### Go Projects

```bash
# Standard Go structure
mkdir -p cmd/myapp internal pkg
touch cmd/myapp/main.go
touch README.md
touch .gitignore

# Initialize go.mod
go mod init github.com/username/myapp

# Common .gitignore for Go
cat > .gitignore << 'EOF'
*.exe
*.exe~
*.dll
*.so
*.dylib
bin/
.env
EOF
```

#### Rust Projects

```bash
# Standard Rust structure (created by cargo)
cargo init --name myapp

# Common .gitignore for Rust
cat > .gitignore << 'EOF'
/target
Cargo.lock
.env
EOF
```

### 5. Write code files

Use the Write tool to create source files with proper content. Always include:

1. **README.md** with:
   - Project description
   - Installation instructions
   - Usage examples

2. **Source files** with meaningful initial code

3. **Configuration files** (package.json, pyproject.toml, go.mod, Cargo.toml)

### 6. Create initial commit

```bash
git add -A
git commit -m "Initial project structure

Generated by NanoClaw code-generator

Co-Authored-By: NanoClaw Agent <agent@nanoclaw.local>"
```

## Best Practices

1. **Descriptive names**: Use kebab-case for project names (e.g., `my-api-server`, `data-processor`)
2. **README first**: Always include a README with project description and usage
3. **License**: Include a MIT LICENSE file when appropriate
4. **Gitignore**: Always create a .gitignore for the language
5. **Dependencies**: Install minimal dependencies needed to run
6. **Tests**: Create at least one test file as a template
7. **Type safety**: Prefer TypeScript over plain JavaScript when appropriate

## Example: Creating a Node.js CLI Tool

```bash
# 1. Create project
PROJECT_NAME="hello-cli"
mkdir -p "/workspace/extra/code/${PROJECT_NAME}"
cd "/workspace/extra/code/${PROJECT_NAME}"

# 2. Initialize git
git init

mkdir -p src test

touch test/cli.test.ts

# 5. Write files using Write tool
# (agent will use Write tool for actual code content)

# 6. Initial commit
git add -A
git commit -m "Initial commit: hello-cli CLI tool

Co-Authored-By: NanoClaw Agent <agent@nanoclaw.local>"
```

## Example: Creating a Python Package

```bash
# 1. Create project
PROJECT_NAME="mypackage"
mkdir -p "/workspace/extra/code/${PROJECT_NAME}"
cd "/workspace/extra/code/${PROJECT_NAME}"

# 2. Initialize git
git init
git config user.email "agent@nanoclaw.local"
git config user.name "NanoClaw Agent"

# 3. Initialize with uv
uv init --name mypackage
uv venv

# 4. Create structure
mkdir -p src/mypackage tests
touch src/mypackage/__init__.py
touch src/mypackage/__main__.py
touch tests/__init__.py
touch tests/test_mypackage.py

# 5. Add dev dependencies
uv add --dev pytest

# 6. Write files using Write tool

# 7. Initial commit
git add -A
git commit -m "Initial commit: mypackage

Co-Authored-By: NanoClaw Agent <agent@nanoclaw.local>"
```

## Running Python Code

After creating a Python project, you can run it with:

```bash
cd /workspace/extra/code/<project-name>
uv run python src/mypackage/main.py
```

Or run tests:

```bash
uv run pytest
```

## Invocation by Other Agents

Agents running in non-main channels can request code generation by:

```
schedule_task(
  prompt: "Create a Node.js project called 'my-api' with Express...",
  target_group_jid: "<main-group-jid>"

```

2. **Using TeamCreate** if in a group with appropriate permissions
